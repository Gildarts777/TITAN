Figure~\ref{fig:ontology_full} gives the complete relation inventory as an adjacency matrix: rows are source types, columns are target types, and a filled cell marks a typed relation from that source to that target, labeled with its letter code (legend below the matrix; the same nine codes as Figure~\ref{fig:ontology}, plus \texttt{J} for the one relation --- Attack Pattern~$\to$~Data Source --- that originates from a STIX object attribute rather than a native relationship object, and is consequently rarer and one-directional). Data Source and Asset never appear as a source, i.e.\ their rows are empty: both are purely terminal in this schema, reached by reasoning but never a valid step \emph{from}.
